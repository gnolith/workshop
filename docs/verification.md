# Health and semantic verification

Public health verifies D1, Workshop schema, Taproot, Diamond, MCP construction,
and compatibility but returns only safe booleans, versions, and overall status.
Admin diagnostics add table/index presence; they never return SQL, binding IDs,
secrets, auth configuration, or stacks.

The local protected probe handler exercises memory creation, validated task creation,
packet resolution, claim, completion, and archive. Production verifiers should
prefer the ordinary authenticated routes and MCP tools. A complete deployment
gate also creates a Taproot Property and Item, statement and reference, verifies
canonical JSON and SPARQL projection agreement, and cleans up through supported
soft-delete/archive operations.

The package MCP gate initializes, lists required tools, performs harmless reads
and temporary task/knowledge calls, rejects anonymous and insufficient calls,
and uses the official MCP client locally. A deployed MCP endpoint and intended
Codex connection remain downstream Site verification. See
`release-evidence.md`; no live gate is claimed without captured output.
