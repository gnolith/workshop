# Health and semantic verification

Public health verifies D1, Workshop schema, Taproot, Diamond, MCP construction,
and compatibility but returns only safe booleans, versions, and overall status.
Admin diagnostics add table/index presence; they never return SQL, binding IDs,
secrets, auth configuration, or stacks.

The local protected probe handler exercises memory creation, validated task
creation, packet resolution, claim, completion, and archive against injected
services. It is package behavior, not a deployed Site acceptance probe.

The package MCP gate initializes, lists required tools, performs harmless reads
and temporary task/knowledge calls, rejects anonymous and insufficient calls,
and uses the official MCP client locally.

The Codex agent creating a Site owns real Taproot/Diamond projection, deployed
MCP, browser behavior, the intended Codex connection, cleanup policy, and live
acceptance. See `release-evidence.md` for the package evidence boundary.
